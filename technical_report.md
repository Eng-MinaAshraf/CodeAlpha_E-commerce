# التقرير الفني الشامل: مشروع المتجر الإلكتروني (E-Commerce Platform)

## 1. ملخص المشروع (Project Summary)
مشروع متجر إلكتروني متكامل (Full-Stack) يهدف إلى تقديم تجربة تسوق إلكترونية سلسة. يعتمد النظام على بنية "العميل والخادم" (Client-Server)، حيث تم بناء الواجهة الأمامية باستخدام تقنيات الويب الأساسية (Vanilla HTML/CSS/JS) لضمان السرعة والخفة، بينما يعتمد الخادم (Backend) على بيئة Node.js وإطار عمل Express.js. تم دمج النظام مع خدمات Supabase لإدارة قاعدة البيانات والمصادقة، بالإضافة إلى ImgBB لاستضافة الصور.

## 2. هيكلية النظام (Architecture)
يتبنى النظام معمارية متعددة الطبقات (N-Tier Architecture):
- **طبقة العرض (Presentation Layer):** واجهة مستخدم (SPA-like) تعتمد على التنقل بين صفحات HTML المنفصلة ولكن مع جلب البيانات ديناميكياً عبر استدعاءات API (Fetch) باستخدام JavaScript (ES Modules).
- **طبقة واجهة برمجة التطبيقات (API Layer):** خادم Express.js يوفر نقاط نهاية (Endpoints) من نوع RESTful لمعالجة طلبات النظام.
- **طبقة البيانات (Data Layer):** تعتمد على Supabase (PostgreSQL) لتخزين البيانات الأساسية (المنتجات، الطلبات، المستخدمين، فريق التوصيل)، و ImgBB لتخزين الأصول (الصور).

## 3. حزمة التقنيات (Tech Stack)
- **الواجهة الأمامية (Frontend):**
  - HTML5 & CSS3 (متجاوب، يدعم RTL والوضع الداكن).
  - Vanilla JavaScript (ES Modules).
- **الخادم (Backend):**
  - **البيئة والإطار:** Node.js, Express.js.
  - **قاعدة البيانات والمصادقة:** Supabase (PostgreSQL, Supabase Auth).
  - **حفظ الملفات:** Multer (Memory Storage) + ImgBB API.
- **الحماية والأداء (Security & Performance):**
  - Helmet (تأمين ترويسات HTTP).
  - CORS (التحكم في الوصول).
  - Compression (ضغط البيانات لتقليل حجم النقل).
  - Express-Validator (التحقق من صحة المدخلات).

## 4. الميزات الرئيسية (Features)
- **إدارة المستخدمين والمصادقة:** تسجيل حساب جديد، تسجيل الدخول، إدارة الملف الشخصي، ورفع الصورة الشخصية (Avatars).
- **إدارة المنتجات (Catalog):** عرض المنتجات، الفلترة حسب الفئة، البحث، والترتيب بناءً على السعر أو الأحدث.
- **سلة المشتريات (Shopping Cart):** إضافة المنتجات، تعديل الكميات، حذف المنتجات، وتفريغ السلة بالكامل.
- **نظام الطلبات (Order System):** إتمام الطلب (Checkout)، حساب الإجمالي ديناميكياً بشكل آمن على الخادم، متابعة حالة الطلب، وإلغاء الطلبات المعلقة.
- **لوحة تحكم الإدارة (Admin Dashboard):** إحصائيات عامة (عدد المستخدمين، المبيعات، المنتجات)، إدارة المستخدمين، تحديث حالة الطلبات، إدارة المنتجات، وإدارة فريق التوصيل (إضافة، تعديل، مسح، وتعيين الطلبات للمندوبين).
- **بوابة التوصيل (Delivery Portal):** نظام مخصص لمندوبي التوصيل لمتابعة الطلبات المسندة إليهم.

## 5. نقاط القوة التقنية (Technical Strengths)
- **خدمة API المركزية (Centralized API Service):** استخدام فئة `ApiService` في الواجهة الأمامية (`public/js/api.js`) لتوحيد طلبات الشبكة، ومعالجة أخطاء HTTP مركزياً، وإرفاق ترويسات المصادقة (Tokens) تلقائياً في كل طلب.
- **تحميل آمن للصور:** رفع الصور إلى الذاكرة المؤقتة بالخادم باستخدام Multer ثم تحويلها إلى ImgBB برمجياً عبر API، مما يقلل العبء على الخادم وتكاليف الاستضافة ويضمن أمان الملفات.
- **حماية المسارات (Middleware-based Security):** استخدام دوال Middleware متخصصة (`requireAuth`, `requireAdmin`) للتحقق من الصلاحيات قبل تنفيذ الإجراءات.
- **تجربة مستخدم تفاعلية:** استخدام أنظمة الإشعارات (Toasts) وحالات التحميل (Skeleton Loaders) لجعل الواجهة سريعة الاستجابة.

## 7. متغيرات البيئة (Environment Variables)
لضمان عمل النظام بشكل صحيح، تم تكوين المتغيرات التالية في ملف `.env`:
- `PORT`: رقم المنفذ الذي يعمل عليه الخادم (مثال: 3000).
- `SUPABASE_URL`: رابط مشروع Supabase.
- `SUPABASE_ANON_KEY`: المفتاح العام (Anon Key) للتخاطب مع Supabase.
- `IMGBB_API_KEY`: مفتاح الواجهة البرمجية الخاص بخدمة ImgBB لاستضافة الصور.
- `NODE_ENV`: بيئة التشغيل (development أو production).

## 8. تدفق البيانات (Data Flow)
1. يقوم المستخدم بإجراء تفاعل عبر الواجهة الأمامية (مثل إضافة منتج للسلة).
2. تقوم الدالة المختصة باستدعاء كائن `api.cart.add` في `api.js`.
3. يضيف `ApiService` رمز المصادقة (JWT) إلى ترويسة الطلب ويُرسل `fetch` إلى الخادم.
4. يستقبل الخادم (Express) الطلب في مسار `/api/cart`.
5. يتحقق الـ Middleware (`requireAuth`) من صحة الرمز عبر Supabase.
6. في حال التحقق بنجاح، يتم تنفيذ الاستعلام في قاعدة البيانات (`cart_items`) في Supabase.
7. يعود الخادم باستجابة JSON.
8. يستقبل الـ Client الاستجابة ويعرض رسالة نجاح (Toast) ويُحدّث واجهة المستخدم (مثل تغيير رقم السلة).

## 9. الحالة الحالية للمشروع (Current Status)
يعتبر المشروع في مرحلة متقدمة من الاكتمال ومناسب ليكون مشروع تخرج احترافي أو نموذج لمتجر تجاري حقيقي. يوفر النظام جميع العمليات الأساسية للتجارة الإلكترونية بدءاً من تسجيل الدخول وتصفح المنتجات وحتى إتمام الطلبات وإدارتها عبر لوحات تحكم متخصصة (للإدارة وفريق التوصيل)، مع وجود معالجة أخطاء شاملة وهيكلية كود نظيفة (Clean Code Architecture).
