# التقرير الفني الشامل: مشروع المتجر الإلكتروني (E-Commerce Platform)

## 1. ملخص المشروع (Project Summary)
مشروع متجر إلكتروني متكامل (Full-Stack) يهدف إلى تقديم تجربة تسوق إلكترونية سلسة. يعتمد النظام على بنية مرنة تدعم **التشغيل المزدوج**:
1. **نمط الخادم التقليدي (Server Mode):** خادم Node.js مع Express.js للتعامل مع الطلبات وإدارة العمليات.
2. **نمط التشغيل بدون خادم (Serverless Mode):** معترض طلبات برمجية في جهة العميل (Fetch Interceptor) يقوم بتحويل مكالمات API تلقائياً لتتم معالجتها مباشرة عبر كود المتصفح والاتصال المباشر بقاعدة بيانات Supabase.

تم استضافة النسخة الحية للمشروع على منصة **Vercel** عبر الرابط:
**[https://code-alpha-e-commerce-seven.vercel.app/](https://code-alpha-e-commerce-seven.vercel.app/)**

## 2. هيكلية النظام (Architecture)
يتبنى النظام معمارية مبتكرة تدمج نمطين للتشغيل:
- **طبقة العرض والعميل (Presentation & Client Layer):** واجهة مستخدم (SPA-like) تعتمد على الاستدعاءات التفاعلية لجلب البيانات ديناميكياً (AJAX / Fetch) باستخدام JavaScript (ES Modules).
- **معترض الطلبات (Fetch Interceptor):** تم تضمين ميزة اعتراض برمجية للطلبات في جهة العميل بملف `public/js/supabase.js`. يقوم باعتراض أي استدعاء يبدأ بـ `/api/` وتحويل مسار الاستعلام مباشرة إلى جداول Supabase باستخدام مكتبة العميل SDK. هذا يسمح بتشغيل التطبيق بالكامل كواجهة أمامية فقط (Serverless Static Site) على منصات الاستضافة الثابتة مثل Vercel.
- **طبقة واجهة برمجة التطبيقات (API Layer):** خادم Express.js (اختياري في النمط السحابي) يوفر نقاط نهاية RESTful.
- **طبقة البيانات (Data Layer):** تعتمد على Supabase (PostgreSQL) لتخزين البيانات (المنتجات، الطلبات، المستخدمين، فريق التوصيل)، و ImgBB لاستضافة صور المنتجات والملفات الشخصية.

## 3. حزمة التقنيات (Tech Stack)
- **الواجهة الأمامية والتشغيل السحابي (Frontend & Serverless):**
  - HTML5 & CSS3 (متجاوب، يدعم RTL والوضع الداكن، مصمم بواجهة حديثة).
  - Vanilla JavaScript (ES Modules).
  - Supabase JS Client SDK (v2.43.0).
  - استضافة الواجهة الأمامية: Vercel.
- **الخادم الخلفي (Backend - اختياري):**
  - **البيئة والإطار:** Node.js, Express.js.
  - **قاعدة البيانات والمصادقة:** Supabase (PostgreSQL, Supabase Auth).
  - **حفظ الملفات:** Multer (Memory Storage) + ImgBB API.
- **الحماية والأداء (Security & Performance):**
  - Helmet (تأمين ترويسات HTTP).
  - CORS (التحكم في الوصول).
  - Compression (ضغط البيانات لتقليل حجم النقل).
  - Express-Validator (التحقق من صحة المدخلات).

## 4. الميزات الرئيسية (Features)
- **إدارة المستخدمين والمصادقة:** تسجيل حساب جديد، تسجيل الدخول التقليدي، تسجيل الدخول عبر Google OAuth، إدارة الملف الشخصي، ورفع الصورة الشخصية (Avatars).
- **إدارة المنتجات (Catalog):** عرض المنتجات، الفلترة حسب الفئة، البحث، والترتيب بناءً على السعر أو الأحدث.
- **سلة المشتريات (Shopping Cart):** إضافة المنتجات، تعديل الكميات، حذف المنتجات، وتفريغ السلة بالكامل، مع دمج السلة المؤقتة للزوار تلقائياً بعد تسجيل الدخول.
- **نظام الطلبات (Order System):** إتمام الطلب (Checkout)، حساب الإجمالي ديناميكياً بشكل آمن، متابعة حالة الطلب، وإلغاء الطلبات المعلقة.
- **لوحة تحكم الإدارة (Admin Dashboard):** إحصائيات عامة (عدد المستخدمين، المبيعات، المنتجات، الإيرادات)، إدارة المستخدمين والمنتجات، تحديث حالة الطلبات، وإدارة فريق التوصيل (إضافة، تعديل، مسح، وتعيين الطلبات للمندوبين).
- **بوابة التوصيل (Delivery Portal):** نظام مخصص لمندوبي التوصيل لمتابعة الطلبات المسندة إليهم وتغيير حالتها إلى "تم التوصيل" وتغيير كلمات المرور وتقديم طلبات الاستعادة.

## 5. نقاط القوة التقنية (Technical Strengths)
- **معترض الطلبات الذكي (Dynamic Fetch Interceptor):** في ملف `public/js/supabase.js` يتم استبدال الدالة الافتراضية `window.fetch` بدالة مخصصة تقوم باكتشاف نوع الطلب. إذا كان الطلب موجهاً إلى `/api/` أو يحتوي على مسار قاعدة بيانات Supabase، يتم اعتراضه وتحويله إلى كائن `handleMockApi` الذي ينفذ نفس منطق الخادم البرمجي محلياً في المتصفح باستخدام Supabase Client SDK، مما يوفر مرونة استضافة الواجهة الأمامية بدون خادم.
- **خدمة API المركزية (Centralized API Service):** استخدام فئة `ApiService` في الواجهة الأمامية (`public/js/api.js`) لتوحيد طلبات الشبكة، ومعالجة أخطاء HTTP مركزياً، وإرفاق ترويسات المصادقة (Tokens) تلقائياً في كل طلب.
- **تحميل آمن للصور:** رفع الصور إلى الذاكرة المؤقتة بالخادم باستخدام Multer ثم تحويلها إلى ImgBB برمجياً عبر API، وفي وضع الـ Serverless يتم تحويلها مباشرة من جهة العميل لـ ImgBB بشكل آمن، مما يضمن أمان وموثوقية استضافة الصور.
- **حماية المسارات (Middleware-based Security):** استخدام دوال Middleware متخصصة (`requireAuth`, `requireAdmin`) للتحقق من الصلاحيات قبل تنفيذ الإجراءات.

## 7. متغيرات البيئة وإعدادات التوجيه (Environment & Redirect Configurations)
لضمان عمل النظام بشكل صحيح في جميع البيئات:
1. **متغيرات البيئة على الخادم الخلفي (ملف `.env`):**
   - `PORT`: رقم المنفذ (مثال: 3000).
   - `SUPABASE_URL` و `SUPABASE_ANON_KEY`: للربط بـ Supabase.
   - `IMGBB_API_KEY`: مفتاح ImgBB لرفع الصور.
2. **إعدادات التوجيه في Supabase (OAuth Redirects):**
   - يجب تعيين الـ **Site URL** في لوحة تحكم Supabase إلى رابط Vercel:
     `https://code-alpha-e-commerce-seven.vercel.app/`
   - يجب إضافة مسارات التوجيه المسموح بها في قائمة الـ **Redirect URLs** لتفعيل التوجيه التلقائي الآمن بعد تسجيل الدخول بواسطة Google OAuth، مثل:
     - `https://code-alpha-e-commerce-seven.vercel.app/**`
     - `http://localhost:5500/**` (للتطوير المحلي)

## 8. تدفق البيانات (Data Flow)
1. يقوم المستخدم بإجراء تفاعل عبر الواجهة الأمامية (مثل إضافة منتج للسلة).
2. تقوم الدالة المختصة باستدعاء كائن `api.cart.add` في `api.js`.
3. يضيف `ApiService` رمز المصادقة (JWT) إلى ترويسة الطلب ويُرسل `fetch` إلى الخادم.
4. **في وضع الـ Serverless:** تقوم دالة `window.fetch` المعترضة بقطع الطلب وتوجيهه إلى `handleMockApi` في `supabase.js` الذي يقوم بإجراء الاستعلام مباشرة في جداول Supabase ويعيد النتيجة للمتصفح.
5. **في وضع الخادم التقليدي:** يمر الطلب للخادم (Express) ويتحقق الـ Middleware (`requireAuth`) من صحة الرمز عبر Supabase، ثم يُنفذ الاستعلام ويعيد النتيجة للمتصفح.
6. يستقبل الـ Client الاستجابة ويعرض رسالة نجاح (Toast) ويُحدّث واجهة المستخدم.

## 9. الحالة الحالية للمشروع (Current Status)
المشروع مكتمل ومستضاف بنجاح على منصة **Vercel** ويعمل بشكل كامل ومستقر بنمط الـ Serverless. تم التحقق من عمل جميع الوظائف (بما في ذلك الدخول الموحد OAuth، سلة المشتريات، إدارة المنتجات والطلبات، لوحات التحكم للإدارة والمندوبين) بنجاح مما يجعله نموذجاً متكاملاً واحترافياً لمتجر إلكتروني حديث.
